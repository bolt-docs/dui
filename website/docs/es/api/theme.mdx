## configure

```ts
configure(opts: { prefix: string; theme?: DuiTheme })
```

Define un tema de color global. Cada componente toma estos colores automáticamente — no necesitas pasarlos manualmente cada vez.

```ts
configure({
  prefix: 'build',
  theme: {
    success: '#22c55e',
    error:   '#ef4444',
    warning: '#eab308',
    info:    '#3b82f6',
    muted:   '#6b7280',
    accent:  '#22d3ee',
  },
})

success('Build pasado')   // verde
error('Build falló')      // rojo
```

### Slots globales

Estos se aplican a cada componente a menos que exista una sobrescritura más específica:

<Field name="success" type="ColorStyle" default="verde">Estados de éxito — marcas, salida de log, finalización de pasos.</Field>
<Field name="error" type="ColorStyle" default="rojo">Estados de error — cruces, mensajes de fallo.</Field>
<Field name="warning" type="ColorStyle" default="amarillo">Estados de advertencia.</Field>
<Field name="info" type="ColorStyle" default="azul">Elementos informativos.</Field>
<Field name="muted" type="ColorStyle" default="gris">Texto secundario, pistas, detalles atenuados.</Field>
<Field name="accent" type="ColorStyle" default="cian">Color de acento para resaltados y elementos decorativos.</Field>

### ColorStyle

Cada slot acepta o un string simple (solo frente) o un objeto con frente y fondo:

```ts
// Solo frente
success: '#22c55e'

// Frente + fondo
error: { fg: '#ff3333', bg: '#330000' }
```

### Slots de componente

Sobrescribe colores para componentes individuales. Vuelve a los slots globales si no están configurados.

```ts
configure({
  theme: {
    // Fallback global
    success: '#22c55e',

    // Logger-specific override
    logger: {
      success: '#00cc66',
    },

    // Box colors
    box: {
      border: '#334155',
      title:  '#ffffff',
      arrow:  '#22d3ee',
      url:    '#60a5fa',
      hint:   { fg: '#94a3b8', bg: '#1e293b' },
    },
  },
})
```

#### Tabla completa de slots

| Componente | Slots | Fallback global |
|-----------|-------|----------------|
| **logger** | `info`, `warn`, `error`, `success`, `debug` | `info`, `warning`, `error`, `success`, `muted` |
| **box** | `border`, `title`, `arrow`, `url`, `hint`, `label`, `value` | `muted`, `accent`, `accent`, `info`, `muted`, `accent`, `accent` |
| **spinner** | `frame`, `success`, `fail`, `warn`, `info` | `accent`, `success`, `error`, `warning`, `info` |
| **list** | `bullet`, `number`, `check`, `cross` | `muted`, `muted`, `success`, `error` |
| **steps** | `success`, `error`, `running`, `pending`, `detail`, `connector` | `success`, `error`, `info`, `muted`, `muted`, `muted` |
| **divider** | `line` | `muted` |
| **prompt** | `message`, `suffix` | `warning`, `muted` |
| **table** | `header`, `border` | — (negrita), `muted` |
| **progress** | `bar` | `accent` |

### Orden de resolución

Al renderizar, DUI resuelve los colores en este orden:

1. **Por llamada** — opción `colors` explícita pasada a la función
2. **Tema del componente** — `theme.logger.success`, `theme.box.arrow`, etc.
3. **Tema global** — `theme.success`, `theme.warning`, etc.
4. **Default** — fallback hardcodeado (verde para éxito, rojo para error, etc.)

Las sobrescrituras por llamada tienen prioridad total:

```ts
// Usa theme.logger.success
success('Desplegado')

// Sobrescribe con color personalizado (solo esta llamada)
success('Desplegado', { color: '#ff8800' })
```

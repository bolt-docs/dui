## Colors

### Colors not showing

If terminal output appears without any ANSI colors:

- The `NO_COLOR` environment variable is set â€” unset it to re-enable colors.
- The output is piped or redirected (non-TTY). Colors are automatically disabled when `stdout` is not a TTY.
- The runtime does not support true color. DUI detects this and falls back to 16-color ANSI where possible.

### "Unsupported color format"

`parseColor()` and `colorize()` accept only specific formats:

```
âś“ hex:      #ff6600, #f60, #ff660080
âś“ rgb:      rgb(255, 102, 0)
âś“ rgba:     rgba(255, 102, 0, 0.5)
âś“ oklch:    oklch(60% 0.15 250)
âś— hsl, hwb, named CSS colors â€” not supported yet
```

### Invalid hex string

<Callout variant="info">âś… Fixed in v0.1.x â€” now throws a clear error.</Callout>

Previously a hex string like `#GGG` produced `NaN` silently. Now it throws: `Invalid hex color: "#GGG"`.

### applyStyle unknown style

Unknown style names in `applyStyle()` are silently ignored â€” they produce no effect but do not error. Use only: `bold`, `dim`, `italic`, `underline`, `inverse`, `hidden`, `strikethrough`.

## Box

### "Cannot read properties of undefined"

<Callout variant="info">âś… Fixed in v0.1.x â€” gracefully falls back to `"double"`.</Callout>

Passing an invalid `style` (e.g. `style: 'none'`) used to crash. Now it falls back to the default `"double"` style.

### Title appears truncated

If the title is longer than the box width minus 4 characters, it is automatically truncated with `"..."`. Keep titles short or increase the box width.

### Box looks empty

Passing an empty `lines` array renders an empty box. This is valid output â€” provide at least one line for content.

## Logger

### debug() shows nothing

`debug()` only outputs when `DEBUG` or `BOLTDOCS_DEBUG` is set in the environment:

```bash
DEBUG=1 node your-cli.js
```

### error() crashes with circular objects

The optional second argument of `error()` is logged via `console.error`. Passing circular references or very large objects may cause issues. Pass only plain serialisable data.

### RangeError: Invalid count value

<Callout variant="info">âś… Fixed in v0.1.x â€” now clamps to minimum 1.</Callout>

Passing a negative or zero value for `len` used to crash. Now it's clamped to `Math.max(1, len)`.

## Spinner & Progress

### Cursor stays hidden after exit

Both `createSpinner` and `createProgressBar` hide the terminal cursor while active. If the process exits without calling `stop()`, the cursor remains hidden.

<Callout variant="warning">
  Always wrap spinner/progress usage in try/catch/finally and call `stop()` in the `finally` block.
</Callout>

```ts
const spinner = createSpinner('Working...')
spinner.start()
try {
  // do work
} finally {
  spinner.stop('success', 'Done!')
}
```

### No animation â€” static text only

In non-TTY environments (CI, pipes, log files), spinners and progress bars degrade gracefully to static line-by-line output. This is expected behaviour.

### progress bar percentage stuck at 0

If `total` is set to `0` or a negative value, the percentage is always `0`. Always use a positive integer for total.

## Table

### Table exceeds terminal width

If too many columns are present, the table may overflow the terminal width. Reduce the number of columns or increase the terminal width.

### Missing cells render as empty

If a row has fewer cells than headers, the missing cells are rendered as empty strings. This is by design.

## Prompt

### confirm() hangs or throws

<Callout variant="info">âś… Fixed in v0.1.x â€” now resolves with default in non-TTY.</Callout>

If `stdin` or `stdout` is not available (non-TTY, headless process), `confirm()` now resolves with the default value instead of throwing.

### Unexpected default value

If the user presses Enter without typing, the prompt resolves to `defaultVal ?? false`. Pass `default: true` to change this.

## Configuration

### configure() has no effect

`configure()` merges with existing config. If called after other modules have already read the config, those modules may still use the old values. Call `configure()` once at the very start of your CLI entry point.

### Misspelled config keys are silently ignored

<Callout variant="info">âś… Fixed in v0.1.x â€” now warns on unknown keys.</Callout>

Invalid keys in `configure()` now log a warning: `[dui] Unknown config key: "xyz"`. Valid keys are listed.

## Environment

### Colors disabled in CI

Most CI systems set `CI=true` which does **not** disable colors by itself. However, some CI systems set `NO_COLOR` or pipe output. Check your CI environment variables.

### terminalWidth() returns 80

When `process.stdout.columns` is unavailable (non-TTY, pipe), DUI falls back to `80` columns. This is normal.
